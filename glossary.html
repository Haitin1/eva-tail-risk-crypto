金融
对数收益率 Log Return
为什么不用涨跌幅
▼

最常见的涨跌幅算法是 (今天价格 - 昨天价格) / 昨天价格，叫做简单收益率。但量化策略几乎都用对数收益率：

r = ln(今天价格 / 昨天价格) # ln 是自然对数

为什么用对数？三个原因：

①对称性：涨50%再跌50%，简单收益率最终是 -25%，但对数收益率加起来是0——数学上更干净
②累加性：持有多天的对数收益率可以直接相加，不需要连乘
③统计性质更好：对数收益率的分布更接近正态，便于后续统计分析
类比：就像地震的里氏震级用对数表示，这样你可以直接比较和相加，而不需要乘法。
金融
多空策略 Long / Short
双向赚钱的逻辑
▼

做多 (Long)：买入资产，期待它涨。涨了赚钱，跌了亏钱。这是普通投资者熟悉的方式。

做空 (Short)：借入资产并卖出，期待它跌，之后低价买回还给借出方，差价是利润。跌了赚钱，涨了亏钱。

我们的策略是多空中性（Market Neutral）：同时做多"安全"资产和做空"危险"资产。这样：

▸市场整体涨：多头赚得比空头亏的多
▸市场整体跌：空头赚得比多头亏的多
▸目标不是预测市场方向，而是赚取相对表现的差价
类比：你认为可口可乐会跑赢百事可乐，所以买可口可乐、卖空百事可乐。不管饮料股整体涨跌，只要可口可乐比百事可乐表现好，你就赚钱。

回测 Backtesting
策略在历史上赚没赚钱
▼

把一个策略在历史数据上模拟运行，看它如果在过去执行会产生什么结果。

我们的回测框架：

①滚动窗口：每次只用过去504天的数据计算信号，模拟"当时不知道未来"
②调仓：每14天重新计算信号，更新持仓
③交易成本：每次调仓扣除10bps（0.1%）的摩擦成本
重要限制：回测好不等于实盘好。常见陷阱是"过拟合"——参数恰好在历史数据上表现优异，但遇到新数据就失效。我们的网格搜索做了243种参数组合，需要谨惕这个风险。

基点 bps（Basis Points）
交易成本的单位
▼

1 bps = 0.01% = 0.0001

金融里用基点而不是百分比，是因为收益率的差异通常很小，用百分比表达不直观。

10 bps = 0.10% # 我们的交易成本假设：买卖各5bps，合计10bps 100 bps = 1% 50 bps = 0.50%

我们设定 cost_bps=10，意思是每次调仓产生的摩擦成本是交易金额的0.1%。对于高频交易这很低，但对于日线级别策略很合理。

统计
正态分布 vs 厚尾分布
为什么金融危机总是"不应该发生"
▼

正态分布（钟形曲线）假设极端事件极其罕见：3个标准差之外的事件概率只有0.3%。大多数传统风险模型基于这个假设。

厚尾分布：真实金融市场的极端事件比正态分布预测的要频繁得多。BTC在正态假设下，2020年那次-61%的暴跌理论上是"十亿年才发生一次"的事件——但它真实发生了。

正态分布假设厚尾（真实市场）
极端事件频率极低，几乎不会发生相对频繁，需要认真对待
风险建模低估崩盘概率更贴近现实
尾部形状参数γγ = 0（精确正态）γ > 0（越大越厚）
类比：正态分布就像保险公司假设"百年一遇"的洪水真的只有100年才来一次。厚尾分布说："不对，这种洪水可能20年就来一次，你得重新定价。"
统计
相关矩阵 Correlation Matrix
10个币种之间的"联动关系地图"
▼

相关系数衡量两个资产的同向运动程度：

+1完全正相关：A涨B必涨，幅度一样
0不相关：A的涨跌与B完全无关
-1完全负相关：A涨B必跌

10个币种两两之间的相关系数组成一个 10×10 的相关矩阵。我们的数据显示，BTC和ETH的相关系数约0.82——高度正相关，几乎一起涨跌。

这个矩阵是PCA的输入——我们用它来找出"到底有几个独立的驱动力在控制这10个币"。

C = (1/(L-1)) × M × Mᵀ # M 是标准化后的收益率矩阵 (10行 × 504列) # 结果 C 是 10×10 矩阵，对角线全是1.0（自相关）
统计
特征值 & 特征向量
PCA的数学基础（不需要记公式）
▼

对相关矩阵做特征分解，可以找到一组"最能解释数据变化的方向"：

▸特征向量：方向。每个特征向量代表"第K个市场驱动力来自哪些币种的组合"
▸特征值：重要程度。特征值越大，这个方向解释的市场波动就越多
类比：把市场想象成一个嘈杂的餐厅。餐厅里有很多声音——厨房的噪音、客人说话、背景音乐。特征分解就是找出"哪几个声源贡献了大部分噪音"（特征值大的方向），以及"每个声源来自哪个方向"（特征向量）。

在我们的10个加密货币数据中：

第1个特征向量（最大特征值）→ 所有币权重相近 = 全市场整体涨跌
第2个特征向量 → 某些币权重大，某些小 = 特定板块的独立行情
后面的特征向量（小特征值）→ 基本是随机噪音
统计
PCA 主成分分析
把10个相关资产变成几个独立"声道"
▼

PCA（Principal Component Analysis）的目标：把高度相关的多个变量，变成完全不相关的几个"主成分"。

R = Λ^(-1/2) × Uᵀ × M # M: 原始标准化收益率矩阵 (10×504) # U: 特征向量矩阵 (旋转方向) # Λ^(-1/2): 特征值的平方根倒数 (标准化幅度) # R: 旋转后的模态收益率 (K×504)，各行完全不相关

旋转后的每一行 R_k 就是一个市场模态（Mode），代表一种独立的市场驱动力：

Mode 1大盘整体行情（BTC主导的系统性涨跌）
Mode 2某类币种的独立行情（比如DeFi板块独立上涨）
Mode 3+更细分的独立驱动力
为什么要这么做？如果直接对BTC和ETH的收益率做极值分析，结果会被它们之间的0.82相关性污染——你说BTC在极端，ETH大概率也在极端，信息是重复的。PCA分解之后，每个模态是独立的，极值分析才能告诉你真正新的信息。
统计
最大似然估计 MLE
拟合GPD参数的方法
▼

给定一批数据，MLE（Maximum Likelihood Estimation）找到最能"解释"这批数据的参数值。

类比：你看到一枚硬币抛了100次，正面出现70次。MLE的回答是：这枚硬币正面概率最可能是70%，因为在所有可能的概率设定下，70%这个参数让观察到"100次中70次正面"的可能性最大。

在我们的代码里，MLE用于找GPD的 γ（形状参数）和 σ（尺度参数），使得GPD分布最好地拟合我们观察到的那批超额收益数据。

minimize: -log_likelihood(γ, σ | exceedances) # 找到使对数似然函数最大的 γ 和 σ # 我们用 L-BFGS-B 优化算法，约束 γ ∈ [-0.49, 2.0]
🌊 极值理论 EVT
EVT
极值理论 EVT
专门研究"最坏情况"的统计学分支
▼

普通统计学研究"平均情况"——均值、方差、正态分布。极值理论（Extreme Value Theory）专门研究极端情况的概率：最大值、最小值、尾部。

起源：EVT最初是工程领域发展的——堤坝设计师需要知道"百年一遇的洪水有多高"，保险精算师需要知道"最坏的飓风会造成多大损失"。金融领域从1990年代开始引入EVT，正是因为传统统计学严重低估了崩盘风险。

EVT的核心结论：无论原始分布是什么形状，超过一定阈值的极值数据，都会收敛到广义帕累托分布（GPD）——这是一个严格的数学定理，不是假设。

▸就像中心极限定理说"很多独立随机变量的和趋向正态分布"
▸EVT的极值定理说"超过高阈值的超额量趋向GPD"
EVT
超阈值方法 POT
只研究"超出阈值的部分"
▼

POT（Peaks-Over-Threshold）是EVT的主要实现方法：

①设定一个高阈值 u（比如：过去504天收益率的前10%分位数）
②只保留超过阈值的那些观测值（叫做超额值 exceedances）
③用GPD拟合这些超额值，得到尾部参数
超额值 = X - u，当 X > u 时 # 比如 u = 2%，某天收益率是5% # 超额值 = 5% - 2% = 3%

为什么不用Block Maxima（BM）法？BM每个时间块只取一个最大值，大量信息被丢弃。POT利用了所有超过阈值的点，统计效率更高——在我们只有504天数据的情况下这很关键。

Block Maxima (BM)Peaks-Over-Threshold (POT)
数据利用率低：每块只用1个最大值高：所有超阈值点都用
样本量（504天数据）约12-50个数据点约50-100个数据点
适合场景长历史、低频数据短历史、日线数据
EVT
广义帕累托分布 GPD
描述"极端事件有多极端"的分布
▼

GPD（Generalized Pareto Distribution）是拟合超额值的标准分布，只有两个参数：

γ形状参数（尾部指数）：决定尾巴有多厚，是我们最关心的参数
σ尺度参数：决定超额值的典型大小，类似标准差
γ 的值尾部类型极端事件现实例子
γ > 0（Fréchet）重尾/厚尾极端损失无上限，越极端越可能金融市场崩盘、加密暴跌
γ = 0（Gumbel）轻尾极端事件可能但有自然限制温度极值
γ < 0（Weibull）有界尾有最大值，绝不会超过某个上限死亡年龄

我们的加密货币数据中，大多数情况下 γ > 0，确认了加密市场是厚尾的。γ 越大，说明这个市场模态越危险。

EVT
尾部形状参数 γ（gamma）
我们策略的核心信号
▼

γ 是我们整个策略最核心的单个数字。它回答一个问题："这个市场驱动力有多危险？"

直觉理解：想象你在测量一条河的洪水风险。γ高的河，"千年一遇"的洪水比"百年一遇"高出很多倍，极度危险。γ低的河，即使是"千年一遇"也不会比"百年一遇"高多少，相对可控。

在我们的策略里：

高γ这个PCA模态的尾部很厚 → 与这个模态高度相关的加密货币更容易出现极端暴跌 → 做空
低γ这个PCA模态的尾部较薄 → 与这个模态相关的加密货币相对安全 → 做多

每个币的风险得分 = Σ |该币对各模态的暴露度| × 各模态的γ

这就是我们截面信号的计算核心。得分高 = 高危 = 做空；得分低 = 相对安全 = 做多。

EVT
极值聚集指数 θ（Theta）
极端事件会"连续来"吗？
▼

θ（Extremal Index）衡量极端事件的聚集程度：

θ = 1极端事件是独立的，发生一次后不影响下次发生概率 → 可能是反转机会
θ → 0极端事件强烈聚集，发生一次后很可能连续发生 → 不要抄底，顺势而为
类比：地震后余震是θ小的情况——大地震后往往还有一系列余震，你不能说"已经震了，现在安全了"。但天气里的极端温度天通常是孤立事件（θ接近1）——今天热不代表明天也热。

我们用 Ferro-Segers (2003) 估计量计算θ：

θ̂ = 2(Σ Tᵢ)² / ((N-1) × Σ Tᵢ²) # Tᵢ = 相邻两次超阈值事件之间的时间间隔 # N = 超阈值事件的总次数

我们测试后发现：日线数据下θ的估计量不稳定（每个模态只有几十个极值点，样本太少），加进策略反而降低了Sharpe。θ在分钟/秒级高频数据上更有价值。

市场模态 vs 行业模态
PCA分解的结果是什么
▼

对10个加密货币做PCA后，我们取前K=7个模态（modes）：

Mode 1市场因子：10个币的权重几乎均等分布。当Mode 1 极端时，是整个加密市场在动，不是某个币的独立行情。它的负尾部γ就是我们的市场崩盘预警信号。
Mode 2-7特异因子：权重集中在某几个币上。论文原始股票数据中Mode 2对应能源板块；在我们的加密数据中，这些模态对应某些币种之间的独立联动（比如Layer-1 vs Meme coins）。
为什么叫"模态"？借用物理学的概念——弦振动有基频（最低的音，对应Mode 1）和泛音（高次谐波，对应Mode 2, 3...）。市场也有"基本振动模式"和"复合振动模式"。

我们用所有K=7个模态的γ加权来计算每个币的综合风险得分——不只看整体市场，也看各个独立驱动力的贡献。

策略
截面信号 Cross-Sectional Signal
比较的是资产之间，不是时间
▼

量化信号有两大类：

类型问的问题例子
时序信号这个资产明天会涨还是跌？MA金叉、RSI超卖
截面信号（我们用的）在所有资产中，哪个相对更好/更差？尾部风险排名

我们的信号是截面排名：在同一时刻，计算10个加密货币各自的尾部风险得分，然后按得分排序。

▸得分最低的4个（尾部最薄）→ 做多
▸得分最高的4个（尾部最厚）→ 做空

这意味着我们不需要预测市场整体涨跌，只需要预测"相对排名"。这个要求显然更容易满足。

signal_i = -(score_i - mean(scores)) / std(scores) # 负号：得分高（危险）→ 信号值低 → 做空 # 得分低（安全）→ 信号值高 → 做多
策略
Regime Switching 防御模式
我们比SimicX多做的一步
▼

Regime（制度/状态）是指市场处于某种系统性状态。我们识别两种状态：

正常市场平均γ < 0.25：运行标准多空策略，赚取截面alpha
压力市场平均γ ≥ 0.25：平掉所有多仓，只持空头，减少暴露

触发逻辑：当所有PCA模态的平均γ超过阈值0.25，说明市场整体的极端事件风险在上升——这是系统性崩盘的前兆，而不是某个币的独立波动。

类比：就像天气预报系统。平时你正常出门活动（多空策略）；当气压骤降、湿度上升（γ升高）预示暴风雨来临时，你提前收好晾晒的衣服躲进屋里（平多仓转防御）——不是等雨下来才反应。

为什么这跟SimicX拒绝的"Local Tail Regime Switch"不一样？

被拒的监测单个资产的右尾变厚 → 做空这个资产（假设会反转）
我们的监测所有PCA模态的平均风险 → 减少整体多头暴露（不预测方向）
策略
滚动窗口 Rolling Window
模拟"当时不知道未来"
▼

每次计算信号，我们只用过去L=504天的数据，不用未来数据。这叫做"expanding window"或"rolling window"。

t=504: 用第1天到第504天的数据 t=505: 用第2天到第505天的数据 t=506: 用第3天到第506天的数据 ... # 窗口向前滚动，始终只看过去504天

这样做的目的是避免未来数据泄露（Look-ahead Bias）——这是回测中最常见也最致命的错误。如果你在2020年计算信号时"不小心"用了2021年的数据，回测结果会虚假地好看。

我们选择L=504（约2年）是因为：

太短（<252天）：GPD拟合样本不够，γ估计不稳定
太长（>1000天）：包含太旧的数据，无法反映当前市场结构变化
504天约2年，POT提取后有50-100个超额值，GPD拟合稳定且反应及时
📊 绩效指标
指标
Sharpe 比率
最重要的风险调整后收益指标
▼

Sharpe比率 = 年化收益 / 年化波动率（衡量每单位风险换来多少收益）

Sharpe = (年化收益率) / (年化标准差) # 加密市场用365天/年（不像股票用252个交易日） # 年化 = 日均收益 × 365，年化波动 = 日波动 × √365
Sharpe值评级解读
< 0❌ 很差扣除风险后亏损
0 ~ 0.5😐 一般有收益但风险也高
0.5 ~ 1.0✅ 不错良好的风险收益比
> 1.0⭐ 优秀机构级水准

我们的策略 Sharpe = 0.74，达到"不错"等级。BTC买入持有 Sharpe = 0.73，两者相近，但我们的最大回撤小得多。

类比：两个基金都每年赚10%。A基金一路平稳，B基金中途曾跌50%再涨回来。你更愿意买A，即使最终收益一样——Sharpe比率量化的就是这个"平稳程度"。
指标
最大回撤 Max Drawdown
最坏情况下你亏了多少
▼

最大回撤 = 从历史最高点到随后最低点的最大跌幅

MaxDD = min( (当前净值 - 历史最高净值) / 历史最高净值 ) # 永远是负数（表示亏损） # 比如净值从100涨到200再跌到120：MaxDD = (120-200)/200 = -40%

最大回撤是投资者心理承受能力的极限测试：

BTC:-83.9% → 你眼睁睁看着100万缩水到16万，绝大多数人在这个过程中会割肉
我们的策略:-55.5% → 100万最坏缩水到44.5万，仍然很痛，但比BTC少亏28%
为什么这比Sharpe更直观？Sharpe是平均值，感受不到痛苦。最大回撤是你最痛的那一刻——它决定你到底撑不撑得住不割肉。
指标
Calmar 比率
用最大回撤衡量的风险调整收益

胜率 = 当天策略收益为正的天数比例

我们的策略胜率约50.8%——几乎是抛硬币。但这不代表策略没有价值：

▸赚钱的那天，平均赚得多（赢大）
▸亏钱的那天，平均亏得少（输小）

这是典型的非对称收益结构——高质量策略不一定需要高胜率，但需要盈利时的幅度大于亏损时的幅度。

类比：职业扑克玩家的胜率经常低于50%，但他们赢的那把往往比输的那把大得多，长期期望值是正的。

把Sharpe比率放在一个滚动窗口（比如90天）内计算，可以看到策略在不同市场环境下的表现变化。

▸滚动Sharpe持续正值 → 策略在各种市场环境下都有效
▸滚动Sharpe在某段时间急剧下降 → 策略在那段时间失效，需要分析原因

从我们的图表可以看到，2018-2019年和2021-2022年策略的滚动Sharpe有明显差异。在熊市期间，我们的滚动Sharpe通常仍为正值，而BTC买持的滚动Sharpe在熊市中严重为负——这正是防御模式发挥作用的时刻。
